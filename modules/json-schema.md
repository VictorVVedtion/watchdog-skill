# JSON Schema Validation & Field Completion Rules

Defines the evaluation result JSON Schema. Quick Mode uses simplified schema, Deep Mode uses full schema.

---

## Quick Mode Schema (Self-Assessment)

Quick Mode is evaluated by Claude itself, generating internal data structures directly — no JSON parsing involved.
For state file consistency, self-assessment results follow this minimal structure:

```json
{
  "overall_score": 35,
  "dimensions": {
    "stuck": 40,
    "drift": 20,
    "hallucination": 10,
    "context_decay": 55,
    "velocity_drop": 30
  },
  "severity": "WARNING",
  "top_signal": "stuck↗",
  "action": "consider checkpoint",
  "grounding": {
    "gates_checked": 15,
    "gates_failed": 2,
    "active_gates": ["exist", "root_cause"],
    "top_gate_failure": "root_cause: symptom-only fix for TypeError"
  }
}
```

`grounding` field is `null` when grounding is not active. No schema validation needed (data generated by Claude).

## Deep Mode Schema (External AI)

### 1. Required Fields & Defaults

| Field | Type | Range/Enum | Missing Default |
|-------|------|-----------|----------------|
| `overall_score` | number | 0-100 | Derive from weighted dimensions |
| `dimensions` | object | Contains all dimensions | Fill missing with default object |
| `dimensions.<dim>.score` | number | 0-100 | Default `0` |
| `dimensions.<dim>.evidence` | string | Non-empty | Default `"not evaluated"` |
| `severity` | string | HEALTHY/WARNING/CRITICAL/EMERGENCY | Derive from overall_score |
| `recommendations` | array | At least 1 item | Default `["no specific recommendations"]` |
| `should_continue` | boolean | true/false | Derive from severity |
| `suggested_action` | string | continue/pause/restart/refocus/notify_human | Derive from severity |
| `remediation` | array | Fix steps | Default `[]` |

### Dimension List

5 dimensions (backward compatible with 4-dimension input): `stuck, drift, hallucination, context_decay, velocity_drop`

When velocity_drop is missing, auto-fill:
```json
"velocity_drop": {"score": 0, "evidence": "not evaluated (external AI did not return this dimension)"}
```

---

### 2. Value Range Validation

**Score Clamp**: `clamped = max(0, min(100, raw))`

**Severity Derivation**: 0-20=HEALTHY, 21-50=WARNING, 51-80=CRITICAL, 81-100=EMERGENCY

**should_continue**: HEALTHY/WARNING=true, CRITICAL/EMERGENCY=false

**suggested_action**: HEALTHY/WARNING=continue, CRITICAL=pause, EMERGENCY=notify_human

---

### 3. Consistency Check

Weighted scoring (5-dim: 25/25/20/15/15, 4-dim: 30/30/25/15):

When `|overall_score - weighted| > 15`: use weighted result, label `[score calibrated]`.

---

### 4. JSON Extraction Strategy

5-layer extraction flow:
1. Parse as JSON directly
2. Extract ` ```json ... ``` ` code block
3. Extract content between `{` and `}`
4. Schema validation and completion (this document's rules)
5. All fail → use Quick Mode self-assessment result
